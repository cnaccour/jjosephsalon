## Tech Stack
- **Framework:** Astro 5 + React islands
- **Styling:** Tailwind CSS 4 (via Vite plugin, no config file — theming in global.css @theme)
- **Animations:** GSAP + ScrollTrigger + SplitType
- **Scroll:** Lenis (synced to GSAP ticker in Layout.astro)
- **Hosting:** Vercel (free tier)
- **Booking:** Boulevard self-booking overlay (injector.min.js)
- **Forms:** Vercel serverless functions (Astro API routes) + email via Resend or Nodemailer SMTP
- **Fonts:** Space Grotesk (headings) + Instrument Sans (body) — self-hosted in public/fonts/

## Important Notes

### Boulevard Integration
- Boulevard's `injector.min.js` is loaded in Layout.astro via inline script
- Booking is triggered by clicking an anchor with `href="#book-now"`
- Hidden trigger element exists in Layout.astro: `<a href="#book-now" id="boulevard-trigger">`
- `src/lib/boulevard.ts` has the `openBooking()` helper

### Content / Data
- All service pricing, location info, and level descriptions live in `src/data/*.json`
- Markdown content (policies, about) lives in `src/content/` and uses Astro Content Collections
- Content collections are defined in `src/content.config.ts` (NOT inside src/content/)

### Tailwind CSS 4
- There is NO `tailwind.config.js` — all theming is in `src/styles/global.css` using `@theme` directive
- Tailwind is added as a Vite plugin in `astro.config.mjs`, NOT as an Astro integration
